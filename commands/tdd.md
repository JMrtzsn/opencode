Strict Red/Green Test-Driven Development is mandatory for this task.

1. **Red Phase (Test):** Write the automated test for the requested behavior. Do not modify implementation.
2. **Failure Verification:** Run the test suite. You must observe the test failure. Rewrite if it passes immediately.
3. **Green Phase (Implement):** Write the minimal implementation required to make the test pass.
4. **Success Verification:** Run the test suite to confirm success.
